# Configuration

## Environment Variables

All runtime configuration is environment-driven.

| Variable | Scope | Description |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | Client/Server | Backend base API URL. |
| `NEXT_PUBLIC_FIREBASE_API_KEY` | Client | Firebase API key. |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | Client | Firebase auth domain. |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | Client | Firebase project ID. |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | Client | Firebase storage bucket. |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | Client | Firebase messaging sender ID. |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | Client | Firebase app ID. |
| `NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID` | Client | Firebase measurement ID (optional). |
| `NEXT_PUBLIC_LIVEBLOCKS_PUBLIC_KEY` | Client | Liveblocks public key. |
| `LIVEBLOCKS_SECRET_KEY` | Server | Liveblocks secret key for auth route. |
| `AI_PROVIDER` | Server | AI provider selection. |
| `AI_MODEL` | Server | AI model selection. |
| `AI_TEMPERATURE` | Server | AI generation temperature. |
| `AI_MAX_TOKENS` | Server | AI response token cap. |
| `GOOGLE_API_KEY` | Server | Google GenAI credential (provider-specific). |
| `MISTRAL_API_KEY` | Server | Mistral credential (provider-specific). |
| `VERTEX_AI_PROJECT_ID` | Server | Vertex project ID (provider-specific). |
| `VERTEX_AI_LOCATION` | Server | Vertex region (`us-central1` default). |

## Collaboration Room ID Helpers

Liveblocks room IDs are generated by helper functions in `src/lib/liveblocks/config.ts`:

- `getDocumentRoomId(documentId)` -> `document:<documentId>`
- `getWorkspaceDocumentRoomId(workspaceId, documentId)` -> `workspace:<workspaceId>:document:<documentId>`

Use these helpers consistently to avoid room ID mismatches.

## AI Runtime Behavior

- `/api/ai-stream` validates credentials before opening SSE stream.
- Missing credentials return `500` with `AI service not configured`.
- `threadId` is generated when omitted.
