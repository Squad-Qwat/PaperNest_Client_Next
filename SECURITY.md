# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| 0.1.x | :white_check_mark: |
| < 0.1.0 | :x: |

## Reporting A Vulnerability

Do not open public issues for security vulnerabilities.

Please report vulnerabilities privately to:

- [TODO] security contact email

If possible, include:

- Affected component or endpoint
- Steps to reproduce
- Potential impact
- Suggested mitigation

## Response Timeline

- Initial acknowledgement: within 72 hours
- Triage update: within 7 business days
- Remediation timeline: shared after impact assessment

## Disclosure Policy

- Reports are handled confidentially until a fix is ready.
- Coordinated disclosure is preferred.
- Credit can be provided after patch release if requested.
